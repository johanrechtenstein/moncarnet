🚗 TonEntretienAuto - Système d'Échéances Temporelles

Date : Février 2026 | Document : Récapitulatif pour MCD/MLD/MPD

1. Contexte et Problématique

Le système d'alertes initial ne prenait en compte que le kilométrage. Or, dans la réalité :

  • Les constructeurs recommandent : "Vidange tous les 15 000 km OU 1 an (le premier atteint)"
  • Les fluides et pièces se dégradent avec le temps, même sans rouler
  • Certains entretiens sont purement temporels (contrôle technique tous les 2 ans)
  • Un utilisateur roulant peu (5 000 km/an) ne serait jamais alerté
🎯 Solution retenue : Système de double échéance (kilométrique ET temporelle), toutes deux facultatives.

2. Structure de Base de Données

2.1 Table MAINTENANCES (Entretiens)

CREATE TABLE maintenances ( id INT PRIMARY KEY AUTO_INCREMENT, vehicle_id INT NOT NULL, type VARCHAR(100) NOT NULL, date DATE NOT NULL, kilometrage INT NOT NULL, prix DECIMAL(10,2), pieces_utilisees TEXT, -- Intervalles (facultatifs) intervalle_km INT, -- Ex: 15000 intervalle_mois INT, -- Ex: 12 (= 1 an) -- Échéances calculées automatiquement prochaine_echeance_km INT, -- Ex: 65000 prochaine_echeance_date DATE, -- Ex: 2027-01-15 created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP, updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP, FOREIGN KEY (vehicle_id) REFERENCES vehicles(id) ON DELETE CASCADE );

2.2 Table VEHICLES (Véhicules)

CREATE TABLE vehicles ( id INT PRIMARY KEY AUTO_INCREMENT, user_id INT NOT NULL, marque VARCHAR(100) NOT NULL, modele VARCHAR(100) NOT NULL, immatriculation VARCHAR(20), -- Facultatif kilometrage_actuel INT NOT NULL, -- 👈 Important pour le calcul des alertes created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP, updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP, FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE );

2.3 Relations

  • users ➜ vehicles : One-to-Many (un utilisateur possède plusieurs véhicules)
  • vehicles ➜ maintenances : One-to-Many (un véhicule a plusieurs entretiens)

4. Calcul des Échéances

4.1 À la Création de l'Entretien

// Exemple : Vidange du 15/01/2026 à 50 000 km // Utilisateur saisit : intervalle_km = 15000, intervalle_mois = 12 // Calculs automatiques : prochaine_echeance_km = 50000 + 15000 = 65000 prochaine_echeance_date = 2026-01-15 + 12 mois = 2027-01-15 // Stockage en BDD : INSERT INTO maintenances VALUES ( ..., intervalle_km = 15000, intervalle_mois = 12, prochaine_echeance_km = 65000, prochaine_echeance_date = '2027-01-15' );

4.2 Fonction de Calcul de l'État

function calculerEtatEntretien(entretien, kmActuel, dateActuelle) { let couleurKm = 'vert'; let couleurDate = 'vert'; // ===== ALERTE KILOMÉTRIQUE ===== if (entretien.prochaine_echeance_km !== null) { const kmRestant = entretien.prochaine_echeance_km - kmActuel; if (kmRestant <= 0) { couleurKm = 'rouge-pulse'; // Dépassé } else if (kmRestant <= 200) { couleurKm = 'rouge'; // Urgent (< 200 km) } else if (kmRestant <= 2000) { couleurKm = 'orange'; // À prévoir (< 2000 km) } // Sinon reste 'vert' } // ===== ALERTE TEMPORELLE ===== if (entretien.prochaine_echeance_date !== null) { const joursRestants = differenceEnJours( entretien.prochaine_echeance_date, dateActuelle ); if (joursRestants <= 0) { couleurDate = 'rouge-pulse'; // Dépassé } else if (joursRestants <= 30) { couleurDate = 'rouge'; // Urgent (< 1 mois) } else if (joursRestants <= 60) { couleurDate = 'orange'; // À prévoir (< 2 mois) } // Sinon reste 'vert' } // ===== PRENDRE LA PLUS URGENTE ===== const priorites = ['vert', 'orange', 'rouge', 'rouge-pulse']; const indexKm = priorites.indexOf(couleurKm); const indexDate = priorites.indexOf(couleurDate); return priorites[Math.max(indexKm, indexDate)]; }

5. Seuils d'Alertes

5.1 Alertes Kilométriques

🟢 VERT : Plus de 2 000 km avant l'échéance
🟠 ORANGE : Entre 200 et 2 000 km avant l'échéance (environ 2-3 semaines à 1000 km/semaine)
🔴 ROUGE : Moins de 200 km avant l'échéance (environ 1 semaine)
🔴💥 ROUGE PULSÉ : Échéance dépassée (km actuel > prochaine échéance)

5.2 Alertes Temporelles

🟢 VERT : Plus de 2 mois avant l'échéance
🟠 ORANGE : Entre 1 et 2 mois avant l'échéance
🔴 ROUGE : Moins de 1 mois avant l'échéance
🔴💥 ROUGE PULSÉ : Échéance dépassée (date actuelle > prochaine échéance)
Règle finale : L'alerte affichée est la plus urgente des deux.
Exemple : Vidange prévue dans 500 km (🔴) OU dans 6 mois (🟢) → Affichage : 🔴

6. Cas d'Usage Spécifiques

6.1 Entretien avec uniquement échéance temporelle

Exemple : Contrôle technique (obligatoire tous les 2 ans, pas lié au kilométrage)

{ type: 'Contrôle technique', date: '2024-02-17', kilometrage: 45000, // Kilométrage au moment du CT intervalle_km: null, // ❌ Pas d'échéance kilométrique intervalle_mois: 24, prochaine_echeance_km: null, prochaine_echeance_date: '2026-02-17' }

6.2 Entretien avec uniquement échéance kilométrique

Exemple : Changement de pneus (uniquement selon l'usure, pas le temps)

{ type: 'Pneus', date: '2025-05-10', kilometrage: 30000, intervalle_km: 50000, intervalle_mois: null, // ❌ Pas d'échéance temporelle prochaine_echeance_km: 80000, prochaine_echeance_date: null }

6.3 Entretien avec double échéance

Exemple : Vidange (15 000 km OU 12 mois, le premier atteint)

{ type: 'Vidange', date: '2026-01-15', kilometrage: 50000, intervalle_km: 15000, // ✅ Échéance kilométrique intervalle_mois: 12, // ✅ Échéance temporelle prochaine_echeance_km: 65000, prochaine_echeance_date: '2027-01-15' }

6.4 Entretien sans échéance (historique uniquement)

Exemple : Réparation ponctuelle (pas de récurrence prévue)

{ type: 'Réparation embrayage', date: '2025-08-20', kilometrage: 75000, intervalle_km: null, // ❌ Pas de récurrence intervalle_mois: null, // ❌ Pas de récurrence prochaine_echeance_km: null, prochaine_echeance_date: null } // → Pas d'alerte générée, conservé uniquement dans l'historique

8. Logique de Mise à Jour

8.1 Modification du Kilométrage Actuel

Lorsque l'utilisateur met à jour le kilométrage de son véhicule :

  • ❌ Aucune modification en base de données des entretiens
  • ✅ Recalcul en temps réel de tous les états d'alertes
  • ✅ Affichage actualisé immédiatement

8.2 Modification d'un Entretien (sous 24h)

L'utilisateur peut modifier tous les champs, y compris les intervalles :

// Si modification de intervalle_km ou intervalle_mois // → Recalcul automatique des prochaines échéances UPDATE maintenances SET intervalle_km = 12000, -- Modifié de 15000 à 12000 prochaine_echeance_km = kilometrage + 12000, updated_at = NOW() WHERE id = 123;

8.3 Après 24h : Verrouillage

  • ❌ Impossible de modifier l'entretien existant
  • ✅ L'utilisateur peut créer une nouvelle ligne de correction (avec traçabilité)
  • ✅ Ou créer un nouvel entretien avec les nouveaux intervalles

9. Points Importants pour le MCD/MLD/MPD

Cardinalités

  • User → Vehicle : 1,N (un utilisateur possède 1 à N véhicules)
  • Vehicle → Maintenance : 0,N (un véhicule a 0 à N entretiens)

Contraintes

  • intervalle_km et intervalle_mois peuvent être NULL (facultatifs)
  • Si intervalle_km est NULL → prochaine_echeance_km est NULL
  • Si intervalle_mois est NULL → prochaine_echeance_date est NULL
  • kilometrage_actuel dans la table vehicles est obligatoire (NOT NULL)
  • Les échéances sont calculées automatiquement à l'insertion/modification

Index Recommandés

-- Pour optimiser les recherches CREATE INDEX idx_vehicle_id ON maintenances(vehicle_id); CREATE INDEX idx_user_id ON vehicles(user_id); CREATE INDEX idx_prochaine_echeance_date ON maintenances(prochaine_echeance_date); CREATE INDEX idx_prochaine_echeance_km ON maintenances(prochaine_echeance_km);

Triggers Potentiels (Optionnel)

-- Calcul automatique des échéances avant insertion CREATE TRIGGER calculate_echeances_before_insert BEFORE INSERT ON maintenances FOR EACH ROW BEGIN IF NEW.intervalle_km IS NOT NULL THEN SET NEW.prochaine_echeance_km = NEW.kilometrage + NEW.intervalle_km; END IF; IF NEW.intervalle_mois IS NOT NULL THEN SET NEW.prochaine_echeance_date = DATE_ADD(NEW.date, INTERVAL NEW.intervalle_mois MONTH); END IF; END;

10. Récapitulatif pour le Cahier des Charges

MVP - Système d'alertes avec double échéance

Chaque entretien peut définir deux types d'échéances (facultatives) :

  • Échéance kilométrique : kilométrage + intervalle en km
  • Échéance temporelle : date + intervalle en mois

Valeurs par défaut proposées :

  • Vidange : 15 000 km OU 12 mois
  • Distribution : 100 000 km OU 72 mois (6 ans)
  • Contrôle technique : 24 mois uniquement

Calcul de l'état :

  • 🟢 Vert : Plus de 2 000 km OU 2 mois avant échéance
  • 🟠 Orange : Entre 200 km et 2 000 km OU entre 1 et 2 mois
  • 🔴 Rouge : Moins de 200 km OU moins de 1 mois
  • 🔴💥 Rouge pulsé : Échéance dépassée

L'alerte affichée est la plus urgente des deux échéances.

L'utilisateur peut :

  • Activer/désactiver chaque type d'échéance via des checkboxes
  • Modifier les intervalles selon les préconisations constructeur
  • Ne définir aucune échéance (entretien sans alerte, historique seulement)

11. Exemple Complet de Flux de Données

Scénario : Jean ajoute une vidange

1. Saisie utilisateur :

  • Type : Vidange
  • Date : 15/01/2026
  • Kilométrage : 50 000 km
  • Prix : 80€
  • Intervalle km : 15 000 km (coché)
  • Intervalle mois : 12 mois (coché)

2. Calcul back-end :

prochaine_echeance_km = 50000 + 15000 = 65000 prochaine_echeance_date = '2026-01-15' + 12 mois = '2027-01-15'

3. Insertion en BDD :

INSERT INTO maintenances VALUES ( id: 42, vehicle_id: 7, type: 'Vidange', date: '2026-01-15', kilometrage: 50000, prix: 80.00, pieces_utilisees: NULL, intervalle_km: 15000, intervalle_mois: 12, prochaine_echeance_km: 65000, prochaine_echeance_date: '2027-01-15', created_at: '2026-01-15 14:30:00', updated_at: '2026-01-15 14:30:00' );

4. Un mois plus tard (15/02/2026), Jean met à jour son kilométrage à 52 000 km :

UPDATE vehicles SET kilometrage_actuel = 52000 WHERE id = 7;

5. Calcul de l'état (front-end ou back-end) :

kmRestant = 65000 - 52000 = 13000 km → Vert ✅ joursRestants = '2027-01-15' - '2026-02-15' = 334 jours → Vert ✅ État final : 🟢 Vert (les deux sont OK)

6. Six mois plus tard (15/07/2026), kilométrage à 63 500 km :

kmRestant = 65000 - 63500 = 1500 km → Orange ⚠️ joursRestants = '2027-01-15' - '2026-07-15' = 184 jours → Vert ✅ État final : 🟠 Orange (l'échéance km approche)

7. Notification à Jean :

⚠️ Votre Peugeot 206 nécessite une vidange 📍 Dans 1 500 km (65 000 km) 📅 Ou avant le 15/01/2027
